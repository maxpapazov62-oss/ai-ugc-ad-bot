## Git Workflow

This project uses Git with GitHub for version control. **Commit and push regularly throughout all work** — after every meaningful unit of progress, not just at the end. This ensures work is never lost and the project state is always recoverable.

Rules:
- Commit after each logical chunk of work (new feature, bug fix, config change, etc.)
- Always push to `origin/main` immediately after committing
- Use clean, descriptive commit messages that reflect what changed and why
- Stage specific files by name — avoid `git add -A` or `git add .`
- Never let significant work accumulate without a commit
